What It Is

The point of Conflab is not to pretend one model has many personalities. It is to give the Director a real orchestration surface for multiple voices across multiple providers at the same time, plus saved personas, reusable conversation setups, and durable output artifacts.

More than a chat window

Conflab is designed for debates, planning sessions, expert panels, writing rooms, invention sessions, and any situation where one response is not enough. You frame the objective, decide what kinds of minds should be in the room, and then work a panel that can mix models and providers at the same time instead of trapping the whole conversation inside one system.

That means you can use it for invention, science, politics, religion, medicine, strategy, education, writing, and other subjects where the whole value comes from hearing different lenses collide. If you want to explore black holes, multiple dimensions, treatment tradeoffs, or a product concept, different agents with different training can surface things one model by itself usually will not. You can put ChatGPT-style agents, Gemini-style agents, Claude-style agents, and other configured participants into the same directed workspace, stage disagreement on purpose, use saved personas as stable participants, keep current and historical conversations organized on disk, and end with outputs that survive after the session.

  • Assemble a panel with different roles, providers, models, or saved personas.
  • Use a totally free tool with the source code included.
  • Steer the discussion as Director instead of passively accepting one answer.
  • Compare how different providers reason, argue, and contribute inside the same conversation.
  • Generate summaries, logs, and image prompts from the strongest material.
  • Turn the conversation into something you can embed, present, or build into a product.
  • Return later and reopen the exact artifacts that mattered.

Designed Around Outputs

Conflab’s workflow is visible in the interface itself: configure the environment, build the conversation, run the panel, then save what matters. The app is opinionated about keeping that chain intact.

The Workflow Shape

The app already has a clear product story hiding inside the interface. This section makes that shape explicit so the site can explain Conflab without drowning people in feature lists.

01

Set the environment

Choose a root folder, connect your model providers, pick a helper model, and give Conflab a stable place to save the work.

02

Build the room

Create a conversation, define the objective, describe the kinds of participants you want, and assign the panel.

03

Direct the panel

Push the room in parallel, sequential, or free-form mode. Ask for disagreement, synthesis, pressure tests, or deliverables.

04

Keep the artifacts

Save summaries, logs, and images so the conversation becomes a reusable asset rather than a moment that disappears.

From rough question to saved deliverable.

This first-pass local site is intentionally simple under the hood: one HTML page, external assets, and portable example artifacts. That gives you a clean base for GitHub Pages or a real `conflab.ai` deployment later, while still being easy to edit locally right now.
